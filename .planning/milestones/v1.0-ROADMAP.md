# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-02-11
**Phases:** 1-4
**Total Plans:** 7

## Overview

从零搭建一个 Go 单文件离线 Bug 分析工具：先建好 Go 项目脚手架与 CSV 解析管线（Phase 1），再实现核心 KPI 看板与瓶颈/人员分析（Phase 2），然后补齐模块热力图与趋势（Phase 3），最后生成日报文本（Phase 4）。四个阶段完成后，用户导入禅道 CSV 即可在 30 秒内看到质量全貌并复制日报。

## Phases

### Phase 1: Foundation & CSV Pipeline

**Goal**: Users can launch a single executable and import Zentao CSV data ready for analysis
**Depends on**: Nothing (first phase)
**Requirements**: INF-01, INF-02, INF-03, INF-04, CSV-01, CSV-02, CSV-03, CSV-04
**Plans**: 2 plans

Plans:
- [x] 01-01-PLAN.md — Go 项目脚手架：embed.FS、HTTP 服务器、自动打开浏览器、前端仪表盘外壳 (INF-01, INF-02, INF-03)
- [x] 01-02-PLAN.md — CSV 上传管线：编码检测、表头映射、数据验证、前端集成 (CSV-01..04, INF-04)

**Success Criteria**:
1. User double-clicks the executable, default browser opens automatically, and the app is ready within 2 seconds
2. User can select a Zentao CSV file via file picker and see it accepted for processing
3. System correctly parses both GBK and UTF-8 encoded CSV files without user intervention
4. System displays clear Chinese error messages when required columns are missing or data format is invalid

**Key Results**:
- Go project compiles to a single ~9MB executable containing all frontend resources
- HTTP server starts on configurable port (default 18088) with graceful shutdown
- Browser auto-opens 300ms after server start (cross-platform)
- CSV parser auto-detects GBK/UTF-8 encoding, parses 2225 bug records
- 4 test cases covering GBK, UTF-8, missing columns, empty file

**Completed**: 2026-02-11

### Phase 2: Core KPI & Bottleneck Analysis

**Goal**: Users can see the full quality status picture — bug inventory, severity distribution, fix time bottlenecks, and personnel load
**Depends on**: Phase 1
**Requirements**: KPI-01, KPI-02, KPI-03, KPI-04, KPI-05, AGE-01, AGE-02, AGE-03, AGE-04, AGE-05, WRK-01, WRK-02
**Plans**: 3 plans

Plans:
- [x] 02-01-PLAN.md — KPI 卡片（今日/昨日新增修复、库存）+ 严重程度分布图（饼图/柱状图切换、全部/仅新增筛选）(KPI-01..05)
- [x] 02-02-PLAN.md — 修复时效统计（均值/P50/分布桶）+ 未解决 Bug 积压排名表 (AGE-01..05)
- [x] 02-03-PLAN.md — 人员工作负载分布图（激活/总量按指派人）(WRK-01, WRK-02)

**Success Criteria**:
1. User sees today's and yesterday's new/fixed bug counts as KPI cards immediately after CSV import
2. User sees bug inventory and can toggle severity chart between pie/bar and "new bugs only" vs "all bugs"
3. User sees fix time stats (average, P50, distribution buckets) and unresolved bugs ranked by backlog age
4. User sees active and total bug distribution per assignee in chart form

**Key Results**:
- 5 KPI cards: 今日新增, 今日修复, Bug 总数, 激活 Bug, 待验证
- Severity chart with pie/bar toggle and all/new-only filter
- Fix time stats (avg, P50) with 4-bucket distribution chart (0-1d, 2-3d, 4-7d, 7+d)
- Backlog table with severity badges and age color coding
- Per-assignee workload charts (active warm gradient, total cool gradient)

**Completed**: 2026-02-11

## Milestone Summary

**Key Decisions:**
- Go + embed.FS for single-binary distribution (~9MB)
- ECharts 5.x embedded for offline chart rendering
- Native HTML/JS (IIFE pattern) — no framework dependency
- GBK/UTF-8 auto-detection for Zentao CSV compatibility
- Analysis package pattern: types.go + domain files + analyze.go entry point
- strings.Builder for report generation — no template library

**Issues Resolved:**
- CSV file has 2225 records not 50K — multiline reproduction steps inflate raw line count
- Zentao CSV quirks handled via LazyQuotes + FieldsPerRecord=-1

**Issues Deferred:**
- None

**Technical Debt Incurred:**
- All Bug fields stored as strings — date parsing deferred to analysis modules
- No comprehensive test suite beyond CSV parser tests
