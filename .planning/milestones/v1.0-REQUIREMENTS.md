# Requirements Archive: v1.0 MVP

**Archived:** 2026-02-11
**Core Value:** 让 Bug 积压和修复瓶颈一眼可见 — 导入 CSV 后 30 秒内看到质量状态全貌

## v1 Requirements — All Complete

### CSV 导入

- [x] **CSV-01**: User can select and upload a CSV file exported from Zentao via file picker — Phase 1 ✓
- [x] **CSV-02**: System auto-detects GBK/UTF-8 encoding and converts to UTF-8 for processing — Phase 1 ✓
- [x] **CSV-03**: System auto-recognizes Zentao CSV column headers and maps to internal fields — Phase 1 ✓
- [x] **CSV-04**: System shows clear error messages when required fields are missing or data format is invalid — Phase 1 ✓

### 核心看板

- [x] **KPI-01**: User can see today's and yesterday's new bug count — Phase 2 ✓
- [x] **KPI-02**: User can see today's and yesterday's fixed bug count — Phase 2 ✓
- [x] **KPI-03**: User can see bug inventory: total count, active count, pending-verification count — Phase 2 ✓
- [x] **KPI-04**: User can see severity distribution chart with pie/bar chart toggle — Phase 2 ✓
- [x] **KPI-05**: User can switch severity view between "new bugs only" and "all bugs" — Phase 2 ✓

### 瓶颈与时效分析

- [x] **AGE-01**: User can see average fix time for resolved bugs — Phase 2 ✓
- [x] **AGE-02**: User can see P50 (median) fix time for resolved bugs — Phase 2 ✓
- [x] **AGE-03**: User can see fix time distribution breakdown (0-1 day / 2-3 days / 4-7 days / 7+ days) — Phase 2 ✓
- [x] **AGE-04**: System calculates backlog age for each unresolved bug (days since creation) — Phase 2 ✓
- [x] **AGE-05**: User can see unresolved bugs listed in descending order by backlog age — Phase 2 ✓

### 模块与质量分析

- [x] **MOD-01**: User can see total bug count and active bug count per module — Phase 3 ✓
- [x] **MOD-02**: User can see a heatmap of Module × Severity showing bug concentration — Phase 3 ✓
- [x] **MOD-03**: User can see module bug trend over the last 7 and 30 days — Phase 3 ✓

### 人员与负载分析

- [x] **WRK-01**: User can see distribution of currently active bugs by assignee — Phase 2 ✓
- [x] **WRK-02**: User can see distribution of total bugs by assignee — Phase 2 ✓

### 日报辅助输出

- [x] **RPT-01**: System auto-generates daily report text in Markdown and plain text format — Phase 4 ✓
- [x] **RPT-02**: Daily report includes today's new / fixed / net change counts — Phase 4 ✓
- [x] **RPT-03**: Daily report highlights severe bug risks (severity 1-2 with long backlog) — Phase 4 ✓
- [x] **RPT-04**: Daily report includes bottleneck module and personnel overview — Phase 4 ✓
- [x] **RPT-05**: User can one-click copy the generated report to clipboard — Phase 4 ✓

### 基础设施

- [x] **INF-01**: Application compiles to a single executable binary via Go embed — Phase 1 ✓
- [x] **INF-02**: Application auto-opens default browser on startup — Phase 1 ✓
- [x] **INF-03**: Application starts in under 2 seconds — Phase 1 ✓
- [x] **INF-04**: Application handles CSV files with 10,000–50,000 rows without degradation — Phase 1 ✓

## v2 Requirements (Deferred)

These requirements were tracked but not in v1.0 scope:

- **ENH-01**: User can pick a custom reference date instead of system date
- **ENH-02**: User can filter analysis by product (所属产品)
- **ENH-03**: User can export charts as images
- **EXT-01**: Support drag-and-drop CSV import
- **EXT-02**: Support importing multiple CSV files and merging data

## Out of Scope

| Feature | Reason |
|---------|--------|
| 禅道数据库/API 直连 | 离线工具定位，仅 CSV 导入 |
| 多用户协作 | 单人本地工具 |
| 在线数据同步 | 无网络通信 |
| 英文/多语言 UI | 目标用户为中文团队 |
| 移动端适配 | 桌面浏览器使用场景 |
| 缺陷管理功能 | 定位为分析工具，不做流程管理 |
